# 📋 データベース更新手順（初心者向け）

## 🎯 何をするの？

タスク管理に新しい機能を追加するため、データベース（データを保存する場所）に新しい項目を追加します。

**追加される機能：**
- ✅ タスクのステータス（未着手、着手中、決裁中、対応済）
- 💬 進捗コメント（例：「伺書起案中です」）
- 👤 担当者の設定
- 📅 最終更新日時

## ⚠️ 作業前の注意事項

### 🔒 必ずバックアップを取る
**重要：** データが消える可能性があるため、必ず作業前にバックアップを取ってください。

### 🕐 作業のタイミング
- 可能であれば、システムを使っていない時間帯（夜間など）に実施
- 他の人がシステムを使用していないことを確認

## 📝 手順1: Supabaseにログイン

### 1-1. Supabaseダッシュボードを開く
1. ブラウザで https://app.supabase.com/ を開く
2. ログイン情報を入力してログイン

### 1-2. プロジェクトを選択
- 職場管理システムのプロジェクトをクリック

## 🔧 手順2: SQL Editorを開く

### 2-1. SQL Editorを見つける
1. 画面左側のメニューを見る
2. 「SQL Editor」または「SQLエディタ」を探してクリック

### 2-2. SQLを入力する場所を見つける

**パターン1: 大きなテキストボックスが表示されている場合**
- 画面中央に大きな入力欄があれば、そこに直接SQLを入力できます
- この場合は次のステップ3に進んでください

**パターン2: 「新しいクエリ」ボタンがある場合**
- 「New query」「新しいクエリ」「+ New」などのボタンをクリック

**パターン3: ボタンが見つからない場合**
1. 画面上部に「+」マークがないか探す
2. 「Create」「作成」「New」などの文字がないか探す
3. 画面右上の角にボタンがある場合もあります

**パターン4: それでも見つからない場合**
1. 画面を少し下にスクロールしてみる
2. 左側のメニューで「SQL Editor」が選択されているか確認
3. ブラウザを更新（F5キー）してもう一度試す

**📝 重要**: どのパターンでも、最終的にSQLを入力できる大きなテキストボックスが表示されればOKです！

## 💻 手順3: SQLコードを実行

### 3-1. 以下のコードをコピー＆ペースト

**⚠️ 重要**: 以下のSQLコードをコピーする際は、「```sql」と「```」の行は含めないでください！

**❌ 間違い**: ```sqlから```まで全部コピー
**✅ 正解**: -- Step 1から最後のセミコロン(;)までをコピー

**✅ 方法1: コメント付きのコード（推奨）**

```
-- Step 1: 新しい列を追加
ALTER TABLE tasks 
ADD COLUMN IF NOT EXISTS status VARCHAR(20) DEFAULT 'not_started',
ADD COLUMN IF NOT EXISTS status_comment TEXT DEFAULT '',
ADD COLUMN IF NOT EXISTS assigned_to VARCHAR(100) DEFAULT '',
ADD COLUMN IF NOT EXISTS last_updated TIMESTAMP WITH TIME ZONE DEFAULT NOW();
```

**✅ 方法2: コメント無しのコード（エラーが出る場合）**

```
ALTER TABLE tasks 
ADD COLUMN IF NOT EXISTS status VARCHAR(20) DEFAULT 'not_started',
ADD COLUMN IF NOT EXISTS status_comment TEXT DEFAULT '',
ADD COLUMN IF NOT EXISTS assigned_to VARCHAR(100) DEFAULT '',
ADD COLUMN IF NOT EXISTS last_updated TIMESTAMP WITH TIME ZONE DEFAULT NOW();
```

**📝 コピーのコツ:**
1. 「```」の文字は絶対に含めない
2. コメント行で文字化けエラーが出る場合は方法2を使用
3. 最後のセミコロン（;）まで必ず含める

### 3-2. 実行する
1. コードを貼り付けたら、「Run」または「実行」ボタンをクリック
2. 「Success」または成功のメッセージが表示されるのを確認

### 3-3. 追加のSQLを実行（オプション）

パフォーマンス向上のため、以下も実行することをお勧めします：

**Step 2のコード（```は含めずにコピー）:**

```
-- Step 2: インデックスを追加（検索を高速化）
CREATE INDEX IF NOT EXISTS idx_tasks_status ON tasks(status);
CREATE INDEX IF NOT EXISTS idx_tasks_assigned_to ON tasks(assigned_to);
CREATE INDEX IF NOT EXISTS idx_tasks_last_updated ON tasks(last_updated);
```

**Step 3のコード（```は含めずにコピー）:**

```
-- Step 3: 既存データのステータスを設定
UPDATE tasks 
SET status = CASE 
  WHEN completed = true THEN 'completed'
  ELSE 'not_started'
END
WHERE status IS NULL OR status = '';
```

## 🚨 トラブルシューティング

### エラーが出た場合

#### ❌ 「syntax error at or near "```"」エラー
**原因**: コードブロックの記号（```）も一緒にコピーしてしまった
**解決**: 
1. SQLエディタの内容を全て削除
2. 「-- Step 1」から「NOW();」までだけをコピー
3. 「```sql」や「```」は絶対に含めない

#### ❌ 「syntax error at or near "-"」エラー
**原因**: コメント記号が「-」（ハイフン1つ）になっている
**解決**: 
1. SQLエディタの内容を全て削除
2. 以下の正しいコードをコピー（ハイフン2つ「--」で始まる）
3. 文字化けや自動変換に注意してコピー

### 🔧 エラー修正用：正しいSQLコード

エラーが出た場合は、以下の正しいコードを使用してください：

**✅ 正しいコード（そのままコピー&ペースト）:**

ALTER TABLE tasks 
ADD COLUMN IF NOT EXISTS status VARCHAR(20) DEFAULT 'not_started',
ADD COLUMN IF NOT EXISTS status_comment TEXT DEFAULT '',
ADD COLUMN IF NOT EXISTS assigned_to VARCHAR(100) DEFAULT '',
ADD COLUMN IF NOT EXISTS last_updated TIMESTAMP WITH TIME ZONE DEFAULT NOW();

**📝 重要なポイント:**
- コメント行（--で始まる行）は省略しても問題ありません
- 上記のコードは4行で構成されています
- 最後に必ずセミコロン（;）があることを確認
- 文字化けしていないことを確認してから実行

**💡 コピーの確認方法:**
1. コピーしたコードの最初が「ALTER TABLE tasks」で始まっている
2. 最後が「NOW();」で終わっている
3. 「-」（ハイフン1つ）や「```」が含まれていない

#### ❌ 「permission denied」エラー
**原因**: データベースの変更権限がない
**解決**: システム管理者に連絡して権限を確認してもらう

#### ❌ 「column already exists」エラー
**原因**: すでに列が存在している
**解決**: 問題ありません。すでに更新済みです

#### ❌ 「table does not exist」エラー
**原因**: tasksテーブルが見つからない
**解決**: プロジェクトが正しいか確認する

### 確認方法

```
-- 緊急時のみ: 新しい列を削除

DROP COLUMN IF EXISTS status_comment,
DROP COLUMN IF EXISTS assigned_to,
DROP COLUMN IF EXISTS last_updated;
```

## 📚 用語説明

- **SQL**: データベースに命令を出すための言語
- **ALTER TABLE**: テーブル（表）の構造を変更する命令
- **Column（列）**: 表の縦の項目（例：名前、住所など）
- **VARCHAR**: 文字を保存するためのデータ型
- **TEXT**: 長い文章を保存するためのデータ型
- **TIMESTAMP**: 日時を保存するためのデータ型
- **Index（インデックス）**: 検索を早くするための仕組み
