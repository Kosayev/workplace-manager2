# 監視・エラートラッキング設定ガイド

## 概要

このプロジェクトには包括的な監視システムが実装されています。本番環境での安定稼働を確保するため、以下の監視ツールを設定してください。

## 1. Sentryエラートラッキング設定

### 1.1 Sentryアカウント作成
1. [Sentry.io](https://sentry.io) でアカウントを作成
2. 新しいプロジェクトを作成（JavaScript/Browser）
3. DSN (Data Source Name) を取得

### 1.2 DSN設定
`app.js` の19行目にあるDSNを実際の値に置き換えます：

```javascript
dsn: 'https://your-actual-dsn@sentry.io/your-project-id', // 実際のDSNに置き換え
```

### 1.3 エラートラッキング機能
- **自動エラー収集**: JavaScript エラーを自動で収集
- **パフォーマンス監視**: ページロード時間、メモリ使用量
- **カスタムエラー**: 手動でエラーを報告可能
- **プライバシー保護**: 個人情報を自動で除外

## 2. UptimeRobot稼働監視設定

### 2.1 UptimeRobotアカウント作成
1. [UptimeRobot](https://uptimerobot.com) で無料アカウントを作成
2. 新しいモニターを追加
3. Monitor Type: HTTP(s)
4. URL: あなたのVercelアプリURL
5. Monitor interval: 5分（無料プランの制限）

### 2.2 アラート設定
- **Email通知**: ダウンタイム時に自動通知
- **Slack/Discord通知**: Webhook URLを設定可能
- **ステータスページ**: 公開ステータスページの作成可能

## 3. GitHub Actions自動バックアップ

### 3.1 シークレット設定
GitHub リポジトリの Settings > Secrets に以下を追加：

```
SUPABASE_URL: https://your-project.supabase.co
SUPABASE_SERVICE_KEY: your-service-role-key
DISCORD_WEBHOOK_URL: (オプション) Discord通知用
```

### 3.2 バックアップ機能
- **自動実行**: 毎日午前2時（UTC）に実行
- **手動実行**: GitHub Actionsページから手動実行可能
- **データ保持**: 30日間のバックアップを保持
- **失敗通知**: バックアップ失敗時にDiscord通知

## 4. 監視ダッシュボード

### 4.1 設定画面での監視状況確認
アプリの設定画面から以下を確認できます：

- **エラートラッキング状況**: Sentry の動作状況
- **稼働時間**: アプリの連続稼働時間
- **パフォーマンス**: システムの健全性ステータス

### 4.2 利用可能な機能
- **監視状況確認**: 各システムの動作状況を確認
- **エラーテスト**: 意図的にエラーを発生させてテスト
- **ログ確認**: ローカルエラーログの確認

## 5. パフォーマンス監視

### 5.1 自動監視項目
- **ページロード時間**: 5秒以上で警告
- **メモリ使用量**: 90%以上で警告
- **API エラー率**: 5%以上で警告
- **キャッシュ効率**: ヒット率を監視

### 5.2 メトリクス収集
以下のメトリクスがローカルストレージに保存されます：
- ページロード回数
- エラー発生回数
- API 呼び出し回数
- キャッシュヒット率

## 6. 緊急時の対応

### 6.1 エラー発生時
1. Sentry ダッシュボードでエラー詳細を確認
2. 設定画面の「ログ確認」でローカルログを確認
3. 必要に応じて「エラーテスト」で問題の再現を試行

### 6.2 ダウンタイム発生時
1. UptimeRobot からアラートメールを受信
2. Vercel ダッシュボードでデプロイ状況を確認
3. Supabase ダッシュボードでデータベース状況を確認

## 7. 無料プランでの制限

### 7.1 Sentry (開発者プラン)
- 月間エラー: 5,000 イベント
- データ保持: 30日間
- 1プロジェクト

### 7.2 UptimeRobot (無料プラン)
- 50 モニター
- 5分間隔チェック
- 2ヶ月間ログ保持

### 7.3 GitHub Actions
- 月間実行時間: 2,000分
- 並行ジョブ: 20

## 8. 推奨設定

### 8.1 本番環境
- Sentry DSN を実際の値に設定
- UptimeRobot で5分間隔監視
- GitHub Actions で毎日バックアップ

### 8.2 開発環境
- Sentry はローカルホストでは無効化
- エラーログはローカルストレージに保存
- 手動でのテスト実行推奨

## 9. トラブルシューティング

### 9.1 よくある問題
- **Sentry が動作しない**: DSN が正しく設定されているか確認
- **バックアップが失敗**: GitHub Secrets が正しく設定されているか確認
- **監視が機能しない**: 本番環境（localhost以外）でテストしているか確認
